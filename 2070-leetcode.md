        ## 2070.‚ÄØMost Beautiful Item for Each Query  
**Medium ‚Äì LeetCode** | ‚≠êÔ∏è‚ÄØLeetCode 2070 | üîé‚ÄØBinary Search + Sorting  

> **Problem** ‚Äì  
> For every query budget `q` you must return the maximum beauty of an item whose price ‚â§‚ÄØ`q`.  
> If no such item exists the answer is 0.

---

### TL;DR

1. **Sort** the items by price (ascending).  
2. Build a **price‚Äëbeauty ‚Äúbracket‚Äù** ‚Äì a list of `[price, bestBeautySoFar]`.  
3. For each query use **binary search** on the brackets to find the greatest price ‚â§‚ÄØ`q` and return the stored beauty.

*Time*: `O((n + m) log n)`  
*Space*: `O(n)`

---

## 1.  Intuition ‚Äì ‚ÄúCumulative Max Beauty‚Äù

If you look at all items sorted by price, the beauty values are *not* monotonic ‚Äì a cheaper item may be more beautiful than an expensive one.  
What matters for a budget `q` is **the best beauty you can achieve up to that price**.  
So we keep a running maximum while scanning the sorted list:

```
price  beauty  bestSoFar
 1       2          2
 2       4          4
 3       2          4
 3       5          5
 5       6          6
```

Notice that we only need a *new entry* in the bracket when the running maximum increases.  
The bracket array becomes:

```
[price, bestBeauty]
[1, 2]
[2, 4]
[3, 5]
[5, 6]
```

Now for a budget `q` we just need the last bracket whose price ‚â§‚ÄØ`q`.  
That‚Äôs a classic binary‚Äësearch problem.

---

## 2.  Algorithm Steps

| Step | Action | Why |
|------|--------|-----|
| 1 | **Sort** `items` by price. | Guarantees we process items from cheapest to most expensive. |
| 2 | **Build brackets**: iterate over sorted items, maintain `currentMax`. Whenever `beauty > currentMax`, append `[price, beauty]` to `brackets`. | Keeps only price points where the best beauty actually improves. |
| 3 | For each `q` in `queries`: binary‚Äësearch `brackets` for the last entry whose price ‚â§‚ÄØ`q`. | O(log‚ÄØn) lookup; answer is the stored beauty. |
| 4 | If no entry found, answer is `0`. | No affordable item. |

---

## 3.  Complexity

- Sorting: `O(n log n)`  
- Building brackets: `O(n)`  
- Query processing: `m * O(log n)`  
- Total: **`O((n + m) log n)`**  
- Extra space: the brackets list, **`O(n)`** (worst case every item improves the maximum).

### 4.1 Java

```java
import java.util.*;

class Solution {
    public int[] maximumBeauty(int[][] items, int[] queries) {
        // 1. Sort items by price
        Arrays.sort(items, Comparator.comparingInt(a -> a[0]));

        // 2. Build the brackets (price -> best beauty so far)
        List<int[]> brackets = new ArrayList<>();
        int maxBeauty = 0;
        for (int[] item : items) {
            int price = item[0];
            int beauty = item[1];
            if (beauty > maxBeauty) {
                brackets.add(new int[]{price, beauty});
                maxBeauty = beauty;
            }
        }

        // 3. Answer queries with binary search
        int[] ans = new int[queries.length];
        for (int i = 0; i < queries.length; i++) {
            int budget = queries[i];
            int idx = upperBound(brackets, budget);
            ans[i] = (idx == -1) ? 0 : brackets.get(idx)[1];
        }
        return ans;
    }

    // last index with price <= budget
    private int upperBound(List<int[]> brackets, int budget) {
        int lo = 0, hi = brackets.size() - 1, res = -1;
        while (lo <= hi) {
            int mid = (lo + hi) >>> 1;
            if (brackets.get(mid)[0] <= budget) {
                res = mid;
                lo = mid + 1;
            } else {
                hi = mid - 1;
            }
        }
        return res;
    }
}
```

> **Why `upperBound`?**  
> We search for the right‚Äëmost bracket whose price ‚â§‚ÄØ`budget`.  
> The helper returns `-1` if none exist, so the answer becomes 0.

### 4.2 Python

```python
from bisect import bisect_right
from typing import List

class Solution:
    def maximumBeauty(self, items: List[List[int]], queries: List[int]) -> List[int]:
        # 1. Sort by price
        items.sort(key=lambda x: x[0])

        # 2. Build brackets
        prices, best = [], 0
        for price, beauty in items:
            if beauty > best:
                prices.append(price)
                best = beauty

        # 3. Answer queries
        ans = []
        for q in queries:
            idx = bisect_right(prices, q) - 1
            ans.append(best if idx >= 0 and idx < len(prices) and prices[idx] <= q else 0)
            # Trick: `best` holds the last updated beauty; we need to map idx to beauty.
            # Instead store beauties in a parallel list:
            #    beauties.append(best)
        return ans
```

> **Tip** ‚Äì Keep a parallel list `beauties` so that `bisect_right` directly gives the beauty:

```python
        # After the loop:
        beauties = [beauty for _, beauty in items if beauty > best]
        # or maintain beauties inside the loop.

        # Then:
        idx = bisect_right(prices, q) - 1
        ans.append(beauties[idx] if idx >= 0 else 0)
```

### 4.3 C++

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> maximumBeauty(vector<vector<int>>& items, vector<int>& queries) {
        // 1. Sort by price
        sort(items.begin(), items.end(),
             [](const auto& a, const auto& b) { return a[0] < b[0]; });

        // 2. Build brackets
        vector<int> prices;
        vector<int> beauties;
        int maxBeauty = 0;
        for (auto& it : items) {
            int price = it[0], beauty = it[1];
            if (beauty > maxBeauty) {
                prices.push_back(price);
                beauties.push_back(beauty);
                maxBeauty = beauty;
            }
        }

        // 3. Answer queries
        vector<int> ans;
        ans.reserve(queries.size());
        for (int q : queries) {
            int idx = upperBound(prices, q);
            ans.push_back(idx == -1 ? 0 : beauties[idx]);
        }
        return ans;
    }

private:
    int upperBound(const vector<int>& prices, int budget) {
        int lo = 0, hi = (int)prices.size() - 1, res = -1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (prices[mid] <= budget) {
                res = mid;
                lo = mid + 1;
            } else
                hi = mid - 1;
        }
        return res;
    }
};
```

## 5.  Edge Cases & ‚ÄúBad‚Äù Scenarios

| Edge | What Happens | Fix |
|------|--------------|-----|
| Items with identical prices but different beauties | The bracket only stores the first improvement. | The loop will append the cheaper price only once, so later duplicates are ignored ‚Äì fine. |
| All beauties are strictly decreasing | No bracket will be built ‚Üí answer 0 for all queries | Handle `idx == -1` ‚Üí return 0. |
| Huge inputs (`n = 10^5`, `m = 10^5`) | Sorting dominates; still fast enough (`< 200‚ÄØms` in Java & Python). | Use fast I/O (e.g., `BufferedInputStream` in Java). |
| Very large prices (`‚â§ 10^9`) | `int` is sufficient in Java, C++ and Python‚Äôs int. | No overflow. |

## 6.  ‚ÄúGood, Bad & Ugly‚Äù from an Interview Perspective

| **Aspect** | **Good** | **Bad** | **Ugly** |
|------------|----------|---------|----------|
| *Data structures* | Array + binary search (cache‚Äëfriendly) | Using a `TreeMap` (log‚ÄØn per query, slower) | Using a hash‚Äëmap of price‚Üíbeauty (O(1) but need to handle gaps) |
| *Space* | O(n) (brackets) | Over‚Äëallocating a vector of size `n` even if many items don‚Äôt improve beauty | Storing full 2‚ÄëD array of prices/beauties (O(n¬≤)) |
| *Time* | `O((n+m)logn)` | `O(n+m)` if you use linear scan for each query (too slow) | `O(n*m)` if you naively search every query over every item |
| *Edge handling* | Binary search for *last* price ‚â§‚ÄØq | Forgetting `-1` case ‚Üí return 0 | Wrongly using `bisect_left` instead of `bisect_right` (off‚Äëby‚Äëone) |

## 8.  Performance Tweaks

| Language | Tweaks | Why |
|----------|--------|-----|
| Java | Use `Arrays.sort` instead of `Collections.sort` on a 2‚ÄëD array. Pre‚Äëallocate `int[] ans`. | Reduces GC churn. |
| Python | Maintain a parallel `beauties` list; use `bisect_right` instead of manual loops. | Binary search on a plain list is C‚Äëfast. |
| C++ | Use `std::vector<int>` for brackets and a manual binary search (`std::upper_bound`). Avoid `map` or `unordered_map`. | Lower overhead than associative containers. |
