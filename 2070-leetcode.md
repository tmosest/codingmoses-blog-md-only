### 1Ô∏è‚É£ Problem Overview  

> **Problem ID**: 2070  
> **Difficulty**: Medium  
> **Tags**: Sorting | Binary Search | Prefix Max | Interview

You‚Äôre given an array of items, each represented as `[price, beauty]`, and an array of queries.  
For each query `q`, you must answer: *What is the maximum beauty of an item whose price ‚â§ q?*  
If no such item exists, return `0`.

---

### 2Ô∏è‚É£ Constraints  

| Parameter | Range | Notes |
|-----------|-------|-------|
| `items.length`, `queries.length` | 1 ‚Ä¶ 10‚Åµ | Big, but still manageable with O((n+q) log n) |
| `price`, `beauty`, `queries[j]` | 1 ‚Ä¶ 10‚Åπ | 32‚Äëbit signed int is enough in Java/Python, but use 64‚Äëbit in C++ |
| Items can have duplicate prices or beauties | ‚úîÔ∏è | Must handle ties correctly |

---

### 3Ô∏è‚É£ Intuition & Key Idea  

1. **Sort the items by price** ‚Äì this puts all affordable items for a budget in a contiguous block.  
2. **Build a ‚Äúprice ‚Üí best beauty‚Äù map** ‚Äì as we sweep the sorted items, keep the running maximum beauty.  
3. **Answer queries in O(log‚ÄØn)** by binary searching the first price that is ‚â§ the query value.  

This reduces the naive O(n¬∑q) approach (checking every item for every query) to O((n+q)‚ÄØlog‚ÄØn).

---

### 4Ô∏è‚É£ Step‚Äëby‚ÄëStep Algorithm  

1. **Sort** `items` by the first element (`price`).  
2. **Pre‚Äëprocess**:  
   ```text
   maxBeauty = 0
   for each item (price, beauty) in sorted items:
       maxBeauty = max(maxBeauty, beauty)
       if price != lastPriceSeen:
           record (price, maxBeauty)  // keeps the best beauty up to this price
   ```
   This produces two parallel arrays: `prices[]` and `beauties[]`.  
3. **Answer queries**: For each `q` in `queries`, binary search `prices[]` to find the largest index `i` with `prices[i] ‚â§ q`.  
   If no such index, answer `0`; otherwise answer `beauties[i]`.

---

### 5Ô∏è‚É£ Complexity Analysis  

| Operation | Time | Space |
|-----------|------|-------|
| Sorting items | **O(n‚ÄØlog‚ÄØn)** | O(n) |
| Pre‚Äëprocessing | **O(n)** | O(n) |
| Query loop (binary search) | **O(q‚ÄØlog‚ÄØn)** | O(1) |
| **Total** | **O((n+q)‚ÄØlog‚ÄØn)** | **O(n)** |

With `n, q ‚â§ 10‚Åµ`, this comfortably fits within the time limits of LeetCode.

---

### 6Ô∏è‚É£ Edge Cases & Pitfalls  

| Edge Case | Why it matters | Fix |
|-----------|----------------|-----|
| Duplicate prices with different beauties | Binary search may return the wrong beauty if we don‚Äôt keep the **maximum** beauty for that price | Keep a running max and update only when beauty increases |
| All queries smaller than the smallest price | Should return `0` | Binary search returns `-1`; treat as `0` |
| Large integer values (up to 1e9) | Risk of overflow in some languages | Use 64‚Äëbit integers (`long` in Java, `long long` in C++, default int in Python) |

#### 7.1 Java (Java¬†17)

```java
import java.util.*;

public class Solution {
    public int[] maximumBeauty(int[][] items, int[] queries) {
        // 1Ô∏è‚É£ sort items by price
        Arrays.sort(items, Comparator.comparingInt(a -> a[0]));

        // 2Ô∏è‚É£ build prefix max beauty arrays
        int n = items.length;
        int[] prices = new int[n];
        int[] bestBeauties = new int[n];
        int maxBeauty = 0;
        int idx = 0;
        for (int[] it : items) {
            int price = it[0];
            int beauty = it[1];
            maxBeauty = Math.max(maxBeauty, beauty);
            prices[idx] = price;
            bestBeauties[idx] = maxBeauty;
            idx++;
        }

        // 3Ô∏è‚É£ answer each query by binary search
        int[] ans = new int[queries.length];
        for (int i = 0; i < queries.length; i++) {
            int q = queries[i];
            int pos = upperBound(prices, q) - 1; // last index ‚â§ q
            ans[i] = (pos >= 0) ? bestBeauties[pos] : 0;
        }
        return ans;
    }

    // first index > target
    private int upperBound(int[] arr, int target) {
        int l = 0, r = arr.length;
        while (l < r) {
            int m = (l + r) >>> 1;
            if (arr[m] <= target) l = m + 1;
            else r = m;
        }
        return l;
    }
}
```

#### 7.2 Python (Python‚ÄØ3)

```python
from bisect import bisect_right
from typing import List

class Solution:
    def maximumBeauty(self, items: List[List[int]], queries: List[int]) -> List[int]:
        # 1Ô∏è‚É£ sort by price
        items.sort(key=lambda x: x[0])

        # 2Ô∏è‚É£ build prefix max arrays
        prices, beauties = [], []
        max_b = 0
        for price, beauty in items:
            max_b = max(max_b, beauty)
            prices.append(price)
            beauties.append(max_b)

        # 3Ô∏è‚É£ answer queries
        res = []
        for q in queries:
            idx = bisect_right(prices, q) - 1
            res.append(beauties[idx] if idx >= 0 else 0)
        return res
```

#### 7.3 C++ (C++17)

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> maximumBeauty(vector<vector<int>>& items, vector<int>& queries) {
        // 1Ô∏è‚É£ sort items by price
        sort(items.begin(), items.end(),
             [](const auto& a, const auto& b){ return a[0] < b[0]; });

        // 2Ô∏è‚É£ prefix max beauty
        int n = items.size();
        vector<int> prices(n), bestBeauty(n);
        int curMax = 0;
        for (int i = 0; i < n; ++i) {
            curMax = max(curMax, items[i][1]);
            prices[i] = items[i][0];
            bestBeauty[i] = curMax;
        }

        // 3Ô∏è‚É£ answer queries
        vector<int> ans;
        ans.reserve(queries.size());
        for (int q : queries) {
            int idx = upper_bound(prices.begin(), prices.end(), q) - prices.begin() - 1;
            ans.push_back(idx >= 0 ? bestBeauty[idx] : 0);
        }
        return ans;
    }
};
```
