## 1. Problem Recap

You are given two integer arrays of equal length, `nums1` and `nums2`.  
You may pick any sub‚Äëarray `[left‚Ä¶right]` (inclusive) and *swap* the two sub‚Äëarrays between the arrays **once** (or do nothing).  
The score is the larger of the two total sums after the optional swap.  
Return the maximum achievable score.

```text
Input  : nums1 = [60,60,60], nums2 = [10,90,10]
Output : 210  (swap index 1)
```

Constraints: `1 ‚â§ n ‚â§ 10^5`, `1 ‚â§ nums[i] ‚â§ 10^4`.

---

## 2. Why the Na√Øve Brute‚ÄëForce Fails

A brute‚Äëforce solution would enumerate all `O(n^2)` sub‚Äëarrays, compute the swap effect and keep the best.  
Even with `n = 10^5`, that‚Äôs ~10^10 operations ‚Äì far beyond practical limits.  
You need an **O(n)** algorithm.

---

## 3. The Insight

Let:

```
sum1 = Œ£ nums1[i]
sum2 = Œ£ nums2[i]
diff[i] = nums2[i] ‚Äì nums1[i]   // element‚Äëwise difference
```

If we swap sub‚Äëarray `[l‚Ä¶r]`:

```
new_sum1 = sum1 + Œ£ diff[l‚Ä¶r]
new_sum2 = sum2 ‚Äì Œ£ diff[l‚Ä¶r]
```

The **only thing that matters** is the sum of `diff[l‚Ä¶r]`.  
Hence we just need the *maximum* and *minimum* sub‚Äëarray sums of the `diff` array.

- **Maximizing the larger array**: add the **maximum** Œî.  
- **Boosting the smaller array**: subtract the **minimum** Œî (i.e. add its absolute value).

The problem reduces to two classic one‚Äëdimensional maximum/minimum sub‚Äëarray problems, solvable by **Kadane‚Äôs algorithm** in linear time.

---

## 4. The O(n) Algorithm (Kadane + One Pass)

1. Compute `sum1` and `sum2`.  
2. Build `diff[i] = nums2[i] - nums1[i]` on the fly.  
3. Run Kadane twice:
   - **maxDiff**: maximum sub‚Äëarray sum of `diff`.  
   - **minDiff**: minimum sub‚Äëarray sum of `diff` (can be found by running Kadane on `-diff` or using a dual‚Äëtrack approach).  
4. Final answer = `max( sum1 + max(0, maxDiff),  sum2 + max(0, -minDiff) )`.

## 5. Code Walkthrough

### 5.1 Java (LeetCode‚Äëstyle)

```java
class Solution {
    public int maximumScoreOfSplicedArray(int[] nums1, int[] nums2) {
        long sum1 = 0, sum2 = 0;
        int n = nums1.length;
        for (int i = 0; i < n; ++i) {
            sum1 += nums1[i];
            sum2 += nums2[i];
        }

        // diff[i] = nums2[i] - nums1[i]
        long maxDiff = Long.MIN_VALUE, curMax = 0;
        long minDiff = Long.MAX_VALUE, curMin = 0;

        for (int i = 0; i < n; ++i) {
            long d = (long) nums2[i] - nums1[i];

            // Kadane for maximum sub‚Äëarray
            curMax = Math.max(d, curMax + d);
            maxDiff = Math.max(maxDiff, curMax);

            // Kadane for minimum sub‚Äëarray
            curMin = Math.min(d, curMin + d);
            minDiff = Math.min(minDiff, curMin);
        }

        long option1 = sum1 + Math.max(0, maxDiff);
        long option2 = sum2 + Math.max(0, -minDiff);
        return (int) Math.max(option1, option2);
    }
}
```

**Why `long`?**  
The sums can reach `10^5 * 10^4 = 10^9`.  
Adding a difference can push it slightly higher; `long` guarantees no overflow.

### 5.2 Python 3

```python
class Solution:
    def maximumScoreOfSplicedArray(self, nums1: list[int], nums2: list[int]) -> int:
        sum1, sum2 = sum(nums1), sum(nums2)

        max_diff = cur_max = 0
        min_diff = cur_min = 0

        for a, b in zip(nums1, nums2):
            d = b - a

            # Kadane for max
            cur_max = max(d, cur_max + d)
            max_diff = max(max_diff, cur_max)

            # Kadane for min
            cur_min = min(d, cur_min + d)
            min_diff = min(min_diff, cur_min)

        option1 = sum1 + max(0, max_diff)
        option2 = sum2 + max(0, -min_diff)
        return max(option1, option2)
```

**Note**: Python automatically handles big integers, so no extra type juggling is needed.

### 5.3 C++ (Fast & Idiomatic)

```cpp
class Solution {
public:
    int maximumScoreOfSplicedArray(vector<int>& nums1, vector<int>& nums2) {
        long long sum1 = 0, sum2 = 0;
        int n = nums1.size();

        for (int i = 0; i < n; ++i) {
            sum1 += nums1[i];
            sum2 += nums2[i];
        }

        long long maxDiff = LLONG_MIN, curMax = 0;
        long long minDiff = LLONG_MAX, curMin = 0;

        for (int i = 0; i < n; ++i) {
            long long d = static_cast<long long>(nums2[i]) - nums1[i];

            curMax = max(d, curMax + d);
            maxDiff = max(maxDiff, curMax);

            curMin = min(d, curMin + d);
            minDiff = min(minDiff, curMin);
        }

        long long option1 = sum1 + max(0LL, maxDiff);
        long long option2 = sum2 + max(0LL, -minDiff);
        return static_cast<int>(max(option1, option2));
    }
};
```

## 7. The Bad ‚Äì Common Pitfalls

| Pitfall | Why It Happens | How to Fix |
|---------|----------------|------------|
| **Overflow in 32‚Äëbit integers** | Sum can exceed 2‚ÄØ√ó‚ÄØ10^9 in extreme cases. | Use `long` (Java/C++) or `long long` (C++). In Python it‚Äôs automatic. |
| **Two separate passes** | Some implement two Kadane passes, doubling runtime. | Run Kadane *once* while tracking both max and min simultaneously. |
| **Ignoring negative differences** | You might add negative Œî, reducing the score. | Take `max(0, maxDiff)` and `max(0, -minDiff)` to avoid harmful swaps. |
| **Off‚Äëby‚Äëone errors in the loop** | `diff` array mis‚Äëcomputed leads to wrong Œî. | Use `zip(nums1, nums2)` or `for (int i = 0; i < n; ++i)` consistently. |
| **Missing the ‚Äúdo nothing‚Äù option** | You must consider the case where swapping hurts. | The `max(0, ‚Ä¶)` part handles it automatically. |

## 8. The Ugly ‚Äì Less‚ÄëKnown Edge Cases

| Edge Case | Description | Quick Fix |
|-----------|-------------|-----------|
| **All differences are zero** | Arrays identical. | Max/Min Œî will be 0; answer is simply `max(sum1, sum2)`. |
| **All differences negative** | `nums2` strictly smaller at every index. | `maxDiff` will be negative, so you rely on boosting the smaller array via `minDiff`. |
| **All differences positive** | `nums2` strictly larger at every index. | Symmetric to the previous; swap never hurts. |
| **Large intermediate sums** | While computing `curMax + d`, intermediate sum could overflow 32‚Äëbit. | Use `long`/`long long`. |
| **`n == 1`** | Only one possible sub‚Äëarray. | Kadane works fine, but initialise `maxDiff` & `minDiff` with the first difference. |
