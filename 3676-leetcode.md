## 1. Problem Overview

> **Count‚ÄØBowl‚ÄØSubarrays**  
> Given an integer array `nums` of distinct elements, count all subarrays of length ‚â•‚ÄØ3 that satisfy  
> **min(nums[l], nums[r]) > max(nums[l+1‚Ä¶r‚Äë1])**.  
> In other words, the two ends form a ‚Äúbowl‚Äù and every inner element is strictly smaller than both ends.

### Sample

```text
Input : nums = [2,5,3,1,4]
Output: 2   // [3,1,4] and [5,3,1,4]
```

## 2. Brute‚ÄëForce Baseline

A naive triple‚Äënested loop enumerating all subarrays would run in **O(n¬≥)** and is impossible for `n = 10‚Åµ`.  
Even a two‚Äënested loop with a running maximum inside (O(n¬≤)) will time‚Äëout on the worst case.

> **Why it fails**  
> *The inner maximum calculation costs O(length)* ‚Äì repeated many times.  
> With 10‚Åµ elements, that‚Äôs ~10¬π‚Å∞ operations.

## 3. Monotonic Stack Insight ‚Äì The Core Idea

The condition can be re‚Äëphrased:

* For a subarray `[l ‚Ä¶ r]`, the **first** element on the left that is **greater** than `nums[r]` must be **inside** the subarray, and similarly for the right.  
* This translates to *finding, for each index, the next greater element (NGE) to the right* and *the previous greater element (PGE) to the left*.

If the distance between an index and its NGE (or PGE) is at least 2 (i.e., the subarray has length ‚â•‚ÄØ3), then that pair contributes **one** bowl.

**Why a monotonic stack works**

* While scanning from left to right, keep a stack of indices whose values are **decreasing**.  
* When you encounter an element that is **larger** than the stack‚Äôs top, the current index is the NGE for all popped indices.  
* Similarly, scanning from right to left gives all PGE values.

Because each index is pushed and popped at most once, the overall complexity is linear.

## 4. Implementation ‚Äì Java

```java
import java.util.*;

class Solution {
    public long bowlSubarrays(int[] nums) {
        int n = nums.length;
        long res = 0;
        if (n < 3) return 0;

        // ----- next greater (to the right) -----
        int[] next = new int[n];
        Arrays.fill(next, -1);
        Deque<Integer> stack = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
                next[stack.pop()] = i;          // i is the NGE for popped index
            }
            stack.push(i);
        }

        // Count bowls where the left end is the NGE of some index
        for (int l = 0; l < n; l++) {
            int r = next[l];
            if (r != -1 && r - l + 1 >= 3) res++;
        }

        // ----- previous greater (to the left) -----
        int[] prev = new int[n];
        Arrays.fill(prev, -1);
        stack.clear();

        for (int i = n - 1; i >= 0; i--) {
            while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
                prev[stack.pop()] = i;          // i is the PGE for popped index
            }
            stack.push(i);
        }

        // Count bowls where the right end is the PGE of some index
        for (int r = 0; r < n; r++) {
            int l = prev[r];
            if (l != -1 && r - l + 1 >= 3) res++;
        }

        return res;
    }
}
```

### Highlights

* `Deque<Integer>` for an efficient stack (amortised O(1) ops).  
* Two separate scans: one left‚Äëto‚Äëright (next greater), one right‚Äëto‚Äëleft (previous greater).  
* 64‚Äëbit `long` is used because the answer can exceed `int`.

## 5. Implementation ‚Äì Python

```python
class Solution:
    def bowlSubarrays(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 3: return 0

        # ----- next greater -----
        next_greater = [-1] * n
        stack = []
        for i, val in enumerate(nums):
            while stack and nums[stack[-1]] < val:
                idx = stack.pop()
                next_greater[idx] = i
            stack.append(i)

        res = 0
        for l in range(n):
            r = next_greater[l]
            if r != -1 and r - l + 1 >= 3:
                res += 1

        # ----- previous greater -----
        prev_greater = [-1] * n
        stack.clear()
        for i in range(n - 1, -1, -1):
            while stack and nums[stack[-1]] < nums[i]:
                idx = stack.pop()
                prev_greater[idx] = i
            stack.append(i)

        for r in range(n):
            l = prev_greater[r]
            if l != -1 and r - l + 1 >= 3:
                res += 1

        return res
```

*Uses `list` as a stack (`append`/`pop`).  
`int` in Python 3 is unbounded, so no worries about overflow.*

## 6. Implementation ‚Äì C++

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    long long bowlSubarrays(vector<int>& nums) {
        int n = nums.size();
        if (n < 3) return 0;
        long long res = 0;

        // ----- next greater (right) -----
        vector<int> next(n, -1);
        vector<int> st;
        for (int i = 0; i < n; ++i) {
            while (!st.empty() && nums[st.back()] < nums[i]) {
                next[st.back()] = i;   // i is NGE for st.back()
                st.pop_back();
            }
            st.push_back(i);
        }

        for (int l = 0; l < n; ++l) {
            int r = next[l];
            if (r != -1 && r - l + 1 >= 3) res++;
        }

        // ----- previous greater (left) -----
        vector<int> prev(n, -1);
        st.clear();
        for (int i = n - 1; i >= 0; --i) {
            while (!st.empty() && nums[st.back()] < nums[i]) {
                prev[st.back()] = i;   // i is PGE for st.back()
                st.pop_back();
            }
            st.push_back(i);
        }

        for (int r = 0; r < n; ++r) {
            int l = prev[r];
            if (l != -1 && r - l + 1 >= 3) res++;
        }

        return res;
    }
};
```

*`vector<int>` + `vector<int>` for stacks (LIFO) gives the same amortised O(1) per op.*

### Interview Trick: ‚ÄúWhy do we need two scans?‚Äù

*Because each end of a bowl can be the NGE of an inner index or the PGE of an outer index.
