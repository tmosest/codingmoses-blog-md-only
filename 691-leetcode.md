        ## 1.  LeetCode‚ÄØ691 ‚Äì **Stickers to Spell Word**

> **Problem statement (shortened)**  
> You have an unlimited supply of *n* different stickers.  
> Each sticker is a lowercase English word.  
> Using the letters on the stickers (you can cut any letter out of any sticker, rearrange them, and you can reuse a sticker any number of times) you must spell a given *target* string.  
> Return the *minimum* number of stickers required, or **‚Äë1** if it‚Äôs impossible.  

|  |  |
|---|---|
| **n** | 1‚ÄØ‚â§‚ÄØn‚ÄØ‚â§‚ÄØ50 |
| **stickers[i] length** | 1‚ÄØ‚â§‚ÄØ‚â§‚ÄØ10 |
| **target length** | 1‚ÄØ‚â§‚ÄØ‚â§‚ÄØ15 |
| **all characters** | lowercase English |

> **Why it matters**  
> The constraints look small, but the problem is NP‚Äëhard.  
> Interviewers love this problem because it forces you to think about *state representation* (bitmask, string, freq array) and *search* (BFS, DFS + memoization, DP).

## 2.  Solution Overview

The standard ‚Äúeasiest to remember‚Äù approach is a **Breadth‚ÄëFirst Search (BFS)** over *remaining target strings*.

1. **Sort** every sticker and the target alphabetically.  
   This allows the `filter()` operation to run in *O(|target| + |sticker|)* by scanning both sorted strings.
2. Put the sorted target in a queue.  
   Each level of the BFS represents using one more sticker.
3. For every state, try every sticker:  
   *Use `filter()`* ‚Äì remove letters that the sticker can cover.  
   *If nothing remains ‚Üí we‚Äôre done* (current depth + 1 stickers).  
   *Otherwise enqueue the new state* if we haven‚Äôt seen it before.
4. If the queue empties ‚Üí impossible ‚Üí return **‚Äë1**.

The BFS guarantees that the first time we reach an empty string we used the minimal number of stickers.

## 3.  Code

### 3.1  Java ‚Äì BFS (the ‚ÄúGangjig‚Äù solution)

```java
import java.util.*;

class Solution {
    public int minStickers(String[] stickers, String target) {
        int n = stickers.length;

        // Sort each sticker and the target
        target = sortChars(target);
        for (int i = 0; i < n; ++i) stickers[i] = sortChars(stickers[i]);

        Queue<String> q = new LinkedList<>();
        Set<String> visited = new HashSet<>();

        q.offer(target);
        int steps = 0;

        while (!q.isEmpty()) {
            int size = q.size();
            steps++;                     // we are using one more sticker
            while (size-- > 0) {
                String cur = q.poll();
                for (String st : stickers) {
                    String nxt = filter(cur, st);
                    if (nxt.isEmpty()) return steps;
                    if (!nxt.equals(cur) && visited.add(nxt)) {
                        q.offer(nxt);
                    }
                }
            }
        }
        return -1;                      // impossible
    }

    // Remove letters of st from a, both sorted
    private String filter(String a, String b) {
        StringBuilder sb = new StringBuilder();
        int j = 0; // pointer into b
        for (char c : a.toCharArray()) {
            boolean found = false;
            while (j < b.length() && b.charAt(j) <= c) {
                if (b.charAt(j++) == c) { // consume this letter
                    found = true;
                    break;
                }
            }
            if (!found) sb.append(c);
        }
        return sb.toString();
    }

    private String sortChars(String s) {
        char[] arr = s.toCharArray();
        Arrays.sort(arr);
        return new String(arr);
    }
}
```

| **Space** | ‚ÄØ`O( 2^m )`‚ÄØ for visited states (worst‚Äëcase). |

### 3.2  Python ‚Äì Memoised DFS / DP (bit‚Äëmask)

The Python implementation uses a *recursive* memoisation with a **bitmask** of the remaining target characters.  
This is the most concise yet highly efficient approach.

```python
class Solution:
    def minStickers(self, stickers: list[str], target: str) -> int:
        from functools import lru_cache

        # Pre‚Äëprocess stickers into frequency counters
        m = len(target)
        target_letters = list(target)

        # Build a list of sticker masks (bitmask of letters it can supply)
        masks = []
        for st in stickers:
            mask = 0
            for ch in set(st):
                mask |= 1 << (ord(ch) - 97)
            masks.append(mask)

        @lru_cache(None)
        def dfs(state: int) -> int:
            if state == 0:
                return 0
            ans = float('inf')
            # Try each sticker
            for mask in masks:
                # If sticker does not cover any needed letter, skip
                if mask & state == 0:
                    continue
                # Build new state after using this sticker
                new_state = state
                for i, ch in enumerate(target_letters):
                    if state & (1 << i) and (mask & (1 << (ord(ch) - 97))):
                        new_state &= ~(1 << i)
                ans = min(ans, 1 + dfs(new_state))
            return ans

        res = dfs((1 << m) - 1)
        return -1 if res == float('inf') else res
```

**Complexities**

|  |  |
|---|---|
| **Time** | ‚ÄØ`O( 2^m * n )`  (m ‚â§ 15 ‚Üí 32‚ÄØ768 states) |
| **Space** | ‚ÄØ`O( 2^m )` for memoisation + recursion stack |

### 3.3  C++ ‚Äì DP with Bitmask (iterative)

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int minStickers(vector<string>& stickers, string target) {
        int m = target.size();
        int full = (1 << m) - 1;

        // Pre‚Äëcompute sticker masks
        vector<int> masks;
        for (auto& st : stickers) {
            int mask = 0;
            for (char c : st) mask |= 1 << (c - 'a');
            masks.push_back(mask);
        }

        const int INF = 1e9;
        vector<int> dp(1 << m, INF);
        dp[0] = 0;          // no letters needed ‚Üí 0 stickers

        for (int state = 0; state <= full; ++state) {
            if (dp[state] == INF) continue;
            for (int smask : masks) {
                int new_state = state;
                // Try to cover each remaining letter with this sticker
                for (int i = 0; i < m; ++i) {
                    if (!(state & (1 << i))) continue;          // already covered
                    if (smask & (1 << (target[i] - 'a')))
                        new_state &= ~(1 << i);                // remove it
                }
                dp[new_state] = min(dp[new_state], dp[state] + 1);
            }
        }
        return dp[full] == INF ? -1 : dp[full];
    }
};
```

**Complexities**

|  |  |
|---|---|
| **Time** | ‚ÄØ`O( 2^m * n * m )` ‚Äì with *m*‚ÄØ‚â§‚ÄØ15 still fast |
| **Space** | ‚ÄØ`O( 2^m )` |

#### 3Ô∏è‚É£ The Good ‚Äì Intuition & What Works

1. **Sort to simplify matching**  
   Sorting stickers and the target lets you run `filter()` in linear time.  
   No nested loops over characters ‚Äì just two pointers.

2. **BFS guarantees optimality**  
   Every layer of BFS represents *using one more sticker*.  
   The first time we hit an empty target string is the answer.

3. **State compression with bitmasks**  
   Python‚Äôs and C++‚Äôs DP solutions compress the remaining letters into a single integer (`1 << m`).  
   Only 2^15 = 32,768 states ‚Äì tiny and fast.

4. **Memoisation cuts recursion**  
   In Python, `@lru_cache` stores results for every sub‚Äëproblem, turning an exponential DFS into a manageable search.

---

#### 4Ô∏è‚É£ The Bad ‚Äì Common Pitfalls

| Mistake | Why it fails | Fix |
|---------|--------------|-----|
| **Using a plain queue of strings** | Strings can be long (up to 15 chars) and many duplicates ‚Äì memory blowup. | Use *sorted strings* + a `Set` to track visited. |
| **Re‚Äëcomputing sticker masks each DFS call** | Adds *O(n)* overhead for every recursive step. | Pre‚Äëcompute a bitmask per sticker once. |
| **Ignoring duplicate stickers** | You might waste time exploring the same sticker again. | Deduplicate stickers or use a `Set` of masks. |
| **Wrong base case in DFS** | Returning 0 for non‚Äëempty state leads to infinite loops. | Base case: `state == 0 ‚Üí 0 stickers`. |
| **Not handling impossible cases** | Returning `float('inf')` directly can cause overflow. | After DP, check for `INF` and return -1. |

---

#### 5Ô∏è‚É£ The Ugly ‚Äì When Complexity Goes Wild

If you na√Øvely try to generate **all combinations** of stickers, you‚Äôll create *exponential* search trees:

- Every string state can branch into *n* stickers ‚Üí `O(n^m)` paths.
- Without a smart mask or sorted string trick, you‚Äôll time‚Äëout on larger targets.

**Solution**:  
Keep your *state representation* tight (bitmask or sorted string) and avoid duplicate work.  
That‚Äôs the secret sauce to turning ‚Äúugly‚Äù into ‚Äúefficient‚Äù.
